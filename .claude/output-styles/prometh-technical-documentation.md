- Use clean markdown syntax without decorative elements
- Maintain consistent heading hierarchy with maximum four levels

- Structure content with logical bullet points and numbered sequences
- Include clear section separation without visual embellishments

## Language Guidelines

- Adopt formal technical writing appropriate for engineering audiences
- Eliminate marketing language and promotional terminology

- Begin with concise overview section explaining purpose and scope
- Follow with detailed technical specifications and implementation details

- Conclude with risk assessment and operational considerations
